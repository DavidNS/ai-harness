# Stage 6.5: Explorer Workflow Skeleton

Goal: mount the staged Explorer workflow in v2 without coupling it to the SDD
`EXPLORE_BUNDLE`.

## Actions

- Add an `EXPLORER` strategy with ordered phases:
  - `EXPLORER_INTAKE`;
  - `EXPLORER_DISCOVERY`;
  - `EXPLORER_DECISION`;
  - `EXPLORER_ARTIFACT`;
  - `EXPLORER_REVIEW`;
  - `EXPLORER_DISTILL`.
- Execute those phases through the generic v2 bundle orchestrator.
- Keep worker playbooks, prompts, and capabilities behind the worker resource
  port and file adapter.
- Write runtime artifacts under `explorer/...` only. Do not publish canonical
  `docs/explorer/...` files in this stage.

## Exit Criteria

- `EXPLORER` can complete with scripted/fake providers.
- Every Explorer phase records its artifact through the artifact port.
- Backend contracts, CLI, and tests recognize the new strategy and phases.

## Deferred Behavior

- No rediscovery loop, none-of-above recovery, or phase invalidation yet.
- No canonical publication or knowledge promotion yet.
- No release/git/CI port integration beyond optional runtime-context inputs.
