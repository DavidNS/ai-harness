# AI Harness Explore Phase - Open Source Tooling

_Last verified: 2026-06-29_

## Goal

The Explore phase should collect objective signals from the repository. The LLM should reason over those signals rather than attempting to infer everything directly from source code.

Suggested pipeline:

```text
Explore
├── Repository fingerprint
├── Static analysis
├── Architecture analysis
├── Dependency analysis
├── Git history
├── CI results
├── Test analysis
├── Security analysis
└── AI synthesis
```

---

# 1. Static Analysis

## Semgrep
Website: https://semgrep.dev/
Repository: https://github.com/semgrep/semgrep

Pros
- Open source Community Edition
- Semantic pattern matching
- Custom rules
- JSON and SARIF output
- 30+ languages
- Runs in GitHub Actions

Best use:
- coding standards
- architectural rules
- anti-pattern detection
- security checks

Notes:
- Excellent fit for an AI harness because rules are easy to write.

---

## SonarQube Community
Website:
https://www.sonarsource.com/products/sonarqube/

Pros
- Code smells
- Technical debt
- Complexity
- Duplication
- Coverage integration

Cons
- Heavy compared with Semgrep.

---

## CodeQL

Website:
https://codeql.github.com/

Pros
- Semantic analysis
- Data-flow analysis
- Call graph queries
- Security analysis

Best use:
- advanced code reasoning
- taint analysis
- finding execution paths

Notes:
- Native GitHub Actions support.
- Open-source engine and queries, although some GitHub Advanced Security features require licensing for private repositories.

# 8. CI Integration

Recommended architecture:

Repository

↓

GitHub Actions

↓

Run tools

↓

Produce artifacts

```text
signals/
    semgrep.json
    codeql.sarif
    dependency.json
    git-history.json
    tests.xml
    coverage.xml
    scorecard.json
```

↓

Harness downloads artifacts

↓

Explore merges signals

↓

AI generates:

- explore.md
- explore.json
- risk assessment
- recommended implementation strategy

---

# Recommended Signal Schema

```json
{
  "tool": "semgrep",
  "category": "architecture",
  "severity": "warning",
  "confidence": 0.98,
  "file": "src/auth.py",
  "message": "Layer violation"
}
```

The AI should consume normalized signals rather than raw tool output.

---

# Overall Recommendation

Core stack:

- Semgrep
- CodeQL
- Code Maat
- Import Linter (Python)
- Madge / Dependency Cruiser
- Trivy
- Gitleaks
- OSV Scanner
- OpenSSF Scorecard

This combination provides strong coverage for architecture, code quality, dependencies, security, and repository health while integrating cleanly into GitHub Actions.
