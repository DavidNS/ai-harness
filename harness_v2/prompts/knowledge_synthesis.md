# Prompt

Create a candidate repository knowledge patch. Propose durable facts about repository files, symbols, APIs, behaviors, constraints, dependencies, conventions, tests, architecture, or documented decisions.

Active claims require repository-backed evidence from repository-relative files. Use run artifacts only as discovery context unless repository evidence supports the claim. Use unverified when evidence is missing or insufficient. Use stable knowledge-source IDs such as claim.domain.001 and do not reuse discovery IDs like C1 or D1. If relations are emitted, their endpoints must reference normalized claim IDs. Create candidate knowledge only; do not claim the patch is accepted source of truth.
